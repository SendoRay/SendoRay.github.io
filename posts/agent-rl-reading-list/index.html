一、Agent RL 核心算法#

1.1 RLHF 与策略优化#

RLHF（Reinforcement Learning from Human Feedback）#

  • 简介：LLM 对齐的标准流水线，分为三阶段——SFT 监督微调、从人类偏好数据训练奖励模型、用 RL（通常是 PPO）对策略进行优化。由 InstructGPT 和 ChatGPT 普及。
  • 关键参考：
    • 论文：Training Language Models to Follow Instructions with Human Feedback (arXiv:2203.02155)
    • OpenAI 博客：Aligning language models to follow instructions
    • DeepMind 博客：Learning through human feedback
    • Anthropic 博客：Training a Helpful and Harmless Assistant with RLHF

PPO（Proximal Policy Optimization）#

  • 简介：RLHF 最常用的 RL 算法。策略梯度方法，使用裁剪代理目标（clipped surrogate objective）确保稳定的同策略更新。在 LLM 场景下将 token 生成视为序列动作，奖励模型提供标量奖励。
  • 关键参考：
    • 原始论文：PPO Algorithms (arXiv:1707.06347)
    • OpenAI Baselines 博客：OpenAI Baselines: PPO
    • 博客解读：PPO: The Key to LLM Alignment
    • 实现细节：The N+ Implementation Details of RLHF with PPO (OpenReview)

GRPO（Group Relative Policy Optimization）#

  • 简介：DeepSeek 提出的 RL 算法，消除了 PPO 中独立的价值模型（critic）。对同一 prompt 采样一组输出，在组内归一化奖励计算相对优势，大幅减少内存和计算开销。用于 DeepSeekMath 和 DeepSeek-R1。
  • 关键参考：
    • DeepSeekMath 论文：arXiv:2402.03300
    • 博客解读：Why GRPO is Important and How it Works
    • 博客解读：GRPO - Deep (Learning) Focus
    • 分析论文：Revisiting GRPO (arXiv:2505.22257)

DAPO（Decoupled Clip and Dynamic Sampling Policy Optimization）#

  • 简介：ByteDance Seed 的开源 RL 系统，在 GRPO 基础上引入四项创新：Clip-Higher（防止熵崩塌）、Dynamic Sampling（过滤简单问题）、Token-level loss（长推理链）、Overlong Reward Shaping（减少截断噪声）。
  • 关键参考：
    • 论文：DAPO (arXiv:2503.14476)
    • 项目主页：dapo-sia.github.io
    • verl 文档：DAPO in verl

Dr. GRPO（Bias-free GRPO）#

  • 简介：GRPO 变体，移除了响应长度归一化和问题级标准差，消除标准 GRPO 中的长度和难度偏差，训练更稳定。
  • 关键参考：
    • 论文：Understanding R1-Zero-Like Training (arXiv:2503.20783)
    • 综述：The State of RL for LLM Reasoning - Sebastian Raschka

On-Policy vs Off-Policy RL for LLMs#

  • 简介：同策略方法（PPO、GRPO）需要从当前策略生成新样本，确保数据分布匹配；离策略方法可从旧数据学习，支持经验回放，样本效率更高。DPO 技术上是离策略的。
  • 关键参考：
    • 分析：LLM Alignment: On-Policy vs. Off-Policy Training Data
    • 深度分析：Online versus Offline RL for LLMs - Cameron Wolfe

Online vs Offline RL for LLMs#

  • 简介：在线 RL（PPO、GRPO）持续从当前策略生成新样本，支持探索；离线 RL 从固定数据集学习，更安全更便宜但可能分布偏移。研究表明在线方法在推理任务上有显著优势。
  • 关键参考：
    • 深度分析：Online versus Offline RL for LLMs
    • 论文：Offline RL for LLM Multi-step Reasoning (ACL 2025)

1.2 偏好优化（无需 RL 的对齐方法）#

DPO（Direct Preference Optimization）#

  • 简介：绕过奖励模型和 RL 循环，直接从偏好数据用闭式损失函数优化策略。基于 Bradley-Terry 模型推导，比 RLHF+PPO 更简单、更稳定，效果相当或更好。
  • 关键参考：
    • 论文：DPO (arXiv:2305.18290)
    • 博客解读：DPO - Deep (Learning) Focus
    • 综述：Comprehensive Survey of DPO (arXiv:2410.15595)

Iterative DPO#

  • 简介：多轮 DPO：每轮 DPO 后模型生成新样本、收集偏好、重新应用 DPO，形成类似在线 RLHF 的自我改进循环。
  • 关键参考：
    • 概览：Iterative DPO - Emergent Mind
    • 论文：Iterative Reasoning Preference Optimization (OpenReview)

Online DPO#

  • 简介：从当前策略在线收集偏好数据（而非固定离线数据集），结合 DPO 损失的简洁性和在线 RL 的探索优势，效果优于离线 DPO。
  • 关键参考：
    • 分析：DPO - Cameron Wolfe
    • 分析：Online versus Offline RL for LLMs

KTO（Kahneman-Tversky Optimization）#

  • 简介：将行为经济学的前景理论应用于 LLM 对齐。与 DPO 需要成对偏好数据不同，KTO 只需二元"好/坏"标签，直接最大化效用，反映损失厌恶等人类认知偏差。
  • 关键参考：
    • 论文：KTO (arXiv:2402.01306)
    • 博客：Better, Cheaper, Faster LLM Alignment with KTO - Contextual AI

IPO（Identity Preference Optimization）#

  • 简介：从一般化 Psi-PO 范式推导，通过设置特定参数为"Identity"创建避免 DPO 在确定性偏好数据上过拟合的方法。由 DeepMind 提出。
  • 关键参考：
    • 论文：A General Theoretical Paradigm to Understand DPO (arXiv:2310.12036)

SLiC-HF（Sequence Likelihood Calibration with Human Feedback）#

  • 简介：基于人类偏好数据用排序损失校准序列似然，无需 RL 训练循环或奖励模型。PPO-based RLHF 的早期替代方案，由 Google 提出。
  • 关键参考：
    • 论文：SLiC-HF (arXiv:2305.10425)

1.3 奖励建模#

Constitutional AI / RLAIF（RL from AI Feedback）#

  • 简介：用 AI 模型（通常是大 LLM）替代人工标注生成偏好标签，依据一组原则（“宪法”）评估输出。Anthropic 的 Constitutional AI 使用自我批评和修订。
  • 关键参考：
    • 论文：Constitutional AI (arXiv:2212.08073)
    • Anthropic PDF：Constitutional AI v2
    • 博客解读：RLAIF - Cameron Wolfe
    • Anthropic 博客：Constitutional AI: Harmlessness from AI Feedback

PRM（Process Reward Model）#

  • 简介：提供步骤级监督的奖励模型，对每个中间推理步骤打分，实现更细粒度的信用分配。OpenAI 的 PRM800K 数据集包含 80 万步级标注。在数学推理上显著优于 ORM。
  • 关键参考：
    • 论文：Let’s Verify Step by Step (arXiv:2305.20050)
    • 数据集：GitHub - openai/prm800k
    • PDF：OpenAI PDF
    • 综述：A Survey of Process Reward Models (arXiv:2510.08049)

ORM（Outcome Reward Model）#

  • 简介：基于最终结果正确性训练的奖励模型，提供稀疏延迟反馈——只判断整个轨迹好坏，不区分哪些步骤正确。训练更简单但反馈粒度粗。
  • 关键参考：
    • 概览：ORM vs PRM - Medium
    • 论文：Exploring the Limit of Outcome Reward (arXiv:2502.06781)

RLVR（Reinforcement Learning with Verifiable Rewards）#

  • 简介：用确定性验证器（代码编译器、数学检查器）替代学习型奖励模型，提供二元正确性反馈。消除奖励黑客风险，降低计算成本。DeepSeek-R1 和 o1 训练的核心。
  • 关键参考：
    • 综述：The State of RL for LLM Reasoning - Sebastian Raschka

1.4 推理增强与自训练#

ReST（Reinforced Self-Training）#

  • 简介：迭代改进 LLM 的方法——生成输出、用奖励模型过滤（growing dataset）、在高质量输出上微调（growing step）。将生成与学习解耦，实现高效离线训练。由 Google DeepMind 提出。
  • 关键参考：
    • 论文：ReST for Language Modeling (arXiv:2308.08998)

STaR（Self-Taught Reasoner）#

  • 简介：通过自举推理能力——为问题生成 CoT 推理链、按正确性过滤、在成功推理链上微调，然后重复。如果模型无法解决，提供提示重试。无需外部监督即可渐进提升推理。
  • 关键参考：
    • 论文：STaR (NeurIPS 2022 / OpenReview)
    • GitHub：github.com/ezelikman/STaR

RFT（Rejection Sampling Fine-Tuning）#

  • 简介：为每个问题生成多个候选解、过滤错误解、在正确样本上微调。比在线 RL 简单（无需奖励模型或 PPO），但通过采样-过滤迭代有效提升推理。也称为 Best-of-N 或 Expert Iteration。
  • 关键参考：
    • 论文：Scaling Relationship on Learning Mathematical Reasoning (arXiv:2308.01825)
    • Kimi k1.5：arXiv:2501.12599
    • 综述：From Rejection Sampling to Expert Iteration (arXiv:2504.11343)

MCTS for LLMs（Monte Carlo Tree Search）#

  • 简介：将 AlphaGo 的 MCTS 规划算法适配到 LLM 推理。LLM 生成多条推理路径（树扩展），由奖励模型或自评估评价，用 UCB 式选择探索有前景的分支。rStar-Math 是典型代表。
  • 关键参考：
    • rStar-Math：arXiv:2501.04519
    • MCTS+偏好学习：arXiv:2405.00451

RL Scaling Laws for LLMs#

  • 简介：研究 RL 训练如何随计算量、模型规模和数据扩展。关键发现包括 RL 计算与推理性能的关系、收益递减 vs 突破性涌现。
  • 关键参考：
    • 分析：RL Scaling Laws for LLMs - Cameron Wolfe
    • 讨论：The New RL Scaling Laws - Nathan Lambert
    • 策展列表：Awesome-RL-for-LRMs - GitHub

1.5 智能体框架与方法#

ReAct（Reasoning + Acting）#

  • 简介：LLM 交替进行推理追踪（CoT）和动作（工具调用、API 调用、搜索）的循环框架。模型推理下一步做什么、执行动作、观察结果、继续推理。
  • 关键参考：
    • 论文：ReAct (arXiv:2210.03629)
    • Google Research 博客：React: Synergizing Reasoning and Acting

Reflexion（Verbal Reinforcement Learning）#

  • 简介：LLM 智能体通过自然语言"言语反馈"迭代改进——每次尝试后生成自我反思存入情景记忆，用于指导未来尝试。将自我反思视为一种言语 RL。
  • 关键参考：
    • 论文：Reflexion (arXiv:2303.11366)
    • GitHub：github.com/noahshinn/reflexion

Toolformer#

  • 简介：教会 LLM 自主决定何时及如何调用外部工具（计算器、QA 检索器、日历等）。模型在自己的训练数据上生成工具调用标注，按有用性过滤，在过滤后的数据上微调。
  • 关键参考：
    • 论文：Toolformer (arXiv:2302.04761)

二、多智能体强化学习（MARL）#

CTDE（Centralized Training, Decentralized Execution）#

  • 简介：MARL 的基础范式——训练时可访问全局信息（集中式），执行时仅用局部观察（分散式）。MADDPG、MAPPO、QMIX、VDN 均遵循此范式。
  • 关键参考：
    • 综述：Introduction to CTDE in MARL (arXiv:2409.03052)

MAPPO（Multi-Agent PPO）#

  • 简介：PPO 在合作多智能体场景下的扩展，使用集中式价值函数（CTDE）。尽管简单，但在标准 MARL 基准上达到 SOTA。
  • 关键参考：
    • 论文：The Surprising Effectiveness of PPO in Cooperative MARL (arXiv:2103.01955)
    • BAIR 博客：MAPPO Blog Post

MADDPG（Multi-Agent Deep Deterministic Policy Gradient）#

  • 简介：Actor-Critic 离策略 MARL 算法，CTDE 范式。每个智能体的 critic 训练时可访问所有智能体的观察和动作，actor 仅用局部观察。支持合作和竞争场景。
  • 关键参考：
    • 论文：MADDPG (arXiv:1706.02275)
    • GitHub：github.com/openai/maddpg

QMIX（Monotonic Value Function Factorisation）#

  • 简介：基于价值的 MARL 算法，使用单调混合网络将联合动作值分解为各智能体效用。扩展 VDN，允许非线性（但单调）的值分解。
  • 关键参考：
    • 论文：QMIX (ICML 2018 / JMLR)

VDN（Value-Decomposition Networks）#

  • 简介：值分解基础方法，将联合值函数表示为各智能体值函数的简单求和，使分散执行可行同时训练使用集中式联合奖励。
  • 关键参考：
    • 论文：VDN (arXiv:1706.05296)

CommNet（Communication Neural Network）#

  • 简介：多智能体架构，通过可微通信通道学习智能体间通信。通信通过跨智能体平均的连续向量实现，梯度可通过通信过程反向传播。
  • 关键参考：
    • 论文：CommNet (arXiv:1605.07736)

TarMAC（Targeted Multi-Agent Communication）#

  • 简介：CommNet 扩展，使用注意力机制允许智能体选择性向特定接收者发送消息，学习"通信什么"和"向谁通信"。
  • 关键参考：
    • 论文：TarMAC (arXiv:1810.11187)

Self-Play in Multi-Agent Learning#

  • 简介：智能体通过与自己（或过去版本）对弈来学习的训练范式，无需外部对手即可持续改进。著名应用包括 AlphaGo Zero、AlphaZero 和 OpenAI Five。
  • 关键参考：
    • 综述：A Survey and Multi-Player MARL Toolbox (arXiv:2212.00253)
    • DeepMind 博客：AlphaGo Zero: Starting from scratch
    • OpenAI 博客：OpenAI Five

LLM-Based Multi-Agent RL#

  • 简介：将 LLM 集成到多智能体 RL 场景的新兴研究方向，LLM 作为能推理、通信和通过交互学习的智能体。
  • 关键参考：
    • 论文：LLM-based MARL: Current and Future Directions (arXiv:2405.11106)
    • Anthropic 博客：How we built our multi-agent research system
    • Anthropic 博客：Building a C compiler with parallel Claudes

三、2024-2025 前沿进展#

DeepSeek-R1#

  • 简介：通过纯 RL（GRPO）激励推理能力的模型，无需人类标注的推理轨迹。多阶段流水线：冷启动 SFT → RL 推理（规则奖励）→ 拒绝采样 & SFT → 最终 RL。发表在 Nature。
  • 关键参考：
    • 论文：DeepSeek-R1 (arXiv:2501.12948)
    • Nature：Nature 论文
    • 训练解析：How DeepSeek R1 was trained - Phil Schmid
    • GRPO 解析：DeepSeek-R1 Dissection: Understanding PPO & GRPO

DeepSeek-R1-Zero#

  • 简介：直接在基座模型上用纯 RL（仅规则奖励）训练的实验，无 SFT 冷启动。推理基准大幅提升但可读性差、语言混合，催生了多阶段 R1 流水线。
  • 关键参考：
    • 同 R1 论文：arXiv:2501.12948

OpenAI o1 / o3 / o4-mini#

  • 简介：OpenAI 的推理模型，通过 RL 后训练发展出自纠正、回溯和多步推理能力。“Competitive Programming with Large Reasoning Models” 论文详述了 RL 在竞赛编程中的扩展。
  • 关键参考：
    • 博客：Learning to reason with LLMs
    • 博客：Introducing o1-preview
    • System Card：o1 System Card
    • 博客：Introducing o3 and o4-mini
    • 博客：o3-mini
    • System Card：o3/o4-mini System Card
    • 论文：Competitive Programming with Large Reasoning Models (arXiv:2502.06807)

Q-Learning for LLMs#

  • 简介：将 Q-learning（基于价值的 RL）应用于 LLM，将 token 生成或推理步骤视为动作，学习价值函数。概念因 OpenAI “Q*” 传闻而受关注。
  • 关键参考：
    • 综述：The State of RL for LLM Reasoning - Sebastian Raschka
    • 文章：Q-Learning for LLMs - Runloop
    • 论文：Empowering LLM-Brained Multi-Tool Reasoner via RL (arXiv:2505.16410)

ReST-MCTS*（MCTS + LLM Self-Training）#

  • 简介：将 MCTS 与过程奖励模型结合实现 LLM 自训练。使用过程奖励引导的树搜索生成高质量推理轨迹，然后微调。NeurIPS 2024。清华大学。
  • 关键参考：
    • NeurIPS：ReST-MCTS* Poster
    • GitHub：github.com/THUDM/ReST-MCTS

SPELL（Self-Play RL for LLMs）#

  • 简介：自博弈 RL 框架，用于演化 LLM 的长篇推理。模型同时生成推理尝试和批评，迭代改进双方。
  • 关键参考：
    • 论文：SPELL (arXiv:2509.23863)

Evolving Self-Play Critic#

  • 简介：推理模型和批评模型之间的对抗博弈，同时演化双方。批评者学习发现推理缺陷，推理者学习产出更难批评的输出。NeurIPS 2025。
  • 关键参考：
    • Poster：NeurIPS 2025

DeepSWE（Coding Agent Trained with RL）#

  • 简介：完全开源的 SOTA 编码智能体，用 RL 端到端训练在软件工程任务上。多轮 RL 训练与工具交互。Together AI / Agentica 发布。
  • 关键参考：
    • 博客：DeepSWE - Together AI
    • 论文：Training Long-Context Multi-Turn SE Agents with RL (arXiv:2508.03501)
    • 模型：HuggingFace - DeepSWE-Preview

SWE-RL（Meta）#

  • 简介：Facebook Research 的 SE 智能体 RL 训练，使用开源软件演化数据和规则奖励。NeurIPS 2025。
  • 关键参考：
    • GitHub：github.com/facebookresearch/swe-rl

Agent-R1（End-to-End Agent RL）#

  • 简介：端到端 RL 训练强大 LLM 智能体的框架，集成多轮工具使用、环境交互和奖励优化到统一 RL 训练流水线。
  • 关键参考：
    • 论文：Agent-R1 (arXiv:2511.14460)

WebRL（Self-Evolving Web Agents）#

  • 简介：通过自演化课程和 Agent-RLAIF 训练 LLM Web 智能体。模型自主生成训练任务、在 Web 环境中尝试、用 AI 反馈改进。ICLR 2024。
  • 关键参考：
    • 论文：WebRL (OpenReview)

AgentGym-RL#

  • 简介：训练 LLM 智能体在多环境中进行长程决策的框架，提供统一的 gym 接口。清华大学。
  • 关键参考：
    • GitHub：github.com/woooodyy/AgentGym-RL

ByteDance Seed-Thinking-v1.5#

  • 简介：Seed-Thinking-v1.5 MoE 推理模型技术详解，包括双轨奖励框架和 RL 训练阶段的算法修改。
  • 关键参考：
    • 博客：Seed-Thinking-v1.5 Technical Details

ByteDance POLARIS#

  • 简介：ByteDance Seed 的 RL 框架，展示定制训练数据和动态温度调整如何让 4B 模型在数学推理上媲美 235B 系统。
  • 关键参考：
    • 博客：POLARIS - 4B Rivals 235B

四、Agent RL 基础设施与框架#

verl / HybridFlow#

  • 简介：ByteDance Seed 发起的灵活高效 RL 后训练框架。支持 PPO、GRPO、DAPO、DrGRPO、REINFORCE++、RLOO、ReMax、SPPO 等。混合控制器编程模型，集成 FSDP、Megatron-LM、vLLM、SGLang。可扩展至 671B 参数。EuroSys 2025。
  • 关键参考：
    • GitHub：github.com/verl-project/verl
    • 论文：HybridFlow (arXiv:2409.19256)
    • 文档：verl.readthedocs.io

OpenRLHF#

  • 简介：基于 Ray 的开源可扩展高性能 RLHF 框架，支持全参数和 LoRA-based RLHF/PPO/DPO 训练 70B+ 模型。提供端到端流水线包括 SFT、奖励建模和 RL 优化。
  • 关键参考：
    • 论文：OpenRLHF (arXiv:2405.11143)
    • GitHub：github.com/openrlhf/openrlhf
    • 文档：openrlhf.readthedocs.io

TRL（Transformers Reinforcement Learning）#

  • 简介：Hugging Face 的 RL 和偏好学习库，原生支持 PPO、DPO、KTO、GRPO 等。集成 HF 生态系统，是轻量级 RLHF 实验的起点。
  • 关键参考：
    • GitHub：github.com/huggingface/trl
    • 文档：TRL Documentation

DeepSpeed-Chat#

  • 简介：微软基于 DeepSpeed 的 RLHF 训练框架，提供三阶段 RLHF 流水线（SFT、奖励模型、RL 微调）。使用 ZeRO 优化，在有限 GPU 上实现大模型 RLHF。
  • 关键参考：
    • 论文：DeepSpeed-Chat (arXiv:2308.01320)
    • GitHub：github.com/microsoft/DeepSpeed

trlX（by CarperAI / EleutherAI）#

  • 简介：大规模 RL 微调框架，支持在线 RL（PPO）和离线 RL（ILQL）。已合并入 TRL 库。
  • 关键参考：
    • 论文：trlX (EMNLP 2023)
    • GitHub：github.com/CarperAI/trlx
    • EleutherAI 博客：trlX Blog

OpenR（Open Reasoner）#

  • 简介：LLM 高级推理开源框架，集成过程奖励模型、MCTS 搜索和 RL 算法（如 GRPO）训练 LLM 复杂推理。
  • 关键参考：
    • 论文：OpenR (arXiv:2410.09671)
    • GitHub：github.com/openreasoner/openr

vLLM for RL Rollouts#

  • 简介：vLLM 高吞吐推理引擎，广泛用于加速 RL 训练的 rollout 阶段（生成模型响应用于奖励计算）。PagedAttention 和连续批处理大幅加速样本生成。
  • 关键参考：
    • 文档：RLHF with vLLM
    • 论文：Active Partial Rollouts in RL (arXiv:2509.18521)

RLlib（Ray）#

  • 简介：基于 Ray 的工业级可扩展 RL 库，提供分布式 RL 训练抽象。原本为传统 RL 设计，已适配 LLM RL 训练工作流。
  • 关键参考：
    • 文档：RLlib - Ray Documentation
    • 论文：RLlib: Abstractions for Distributed RL (PMLR)

AReaL（Large-Scale Asynchronous RL）#

  • 简介：大规模异步 RL 系统，使用异步 rollout 和训练重叠计算，减少 RL 训练中 GPU 空闲时间。
  • 关键参考：
    • 论文：AReaL (arXiv:2505.24298)

SRL（Scaling Distributed RL to 10,000+ GPUs）#

  • 简介：将分布式 RL 扩展到万卡以上的系统，解决大规模 RL 训练的工程挑战。
  • 关键参考：
    • 论文：SRL (OpenReview)

NVIDIA NeMo-RL#

  • 简介：NVIDIA 的 RL 训练框架，Megatron-Core 后端支持原生集成优化 RL 训练吞吐量。支持 GRPO 复现 DeepScaleR recipe。
  • 关键参考：
    • 博客：NeMo-RL Megatron-Core Support
    • 博客：Reproducing DeepScaleR with GRPO
    • 博客：Train a Reasoning LLM in One Weekend

七、关键趋势总结#

1. GRPO 取代 PPO 成为主流#

最大的算法转变是从 PPO（需要昂贵 critic 网络）转向 GRPO 及变体（DAPO、Dr. GRPO），通过组内相对优势消除价值函数，大幅减少内存和计算。DeepSeek-R1 在大规模上验证了此方法。

2. 可验证奖励取代奖励模型（RLVR）#

用确定性验证器（代码编译器、数学检查器、规则评分）替代学习型奖励模型，避免奖励黑客风险并消除昂贵的奖励模型训练步骤。DeepSeek-R1 和 o1 训练的核心。

3. RL 训练编码智能体#

2025 年出现多个系统（DeepSWE、SWE-RL）用 RL 端到端训练编码智能体处理真实软件工程任务，超越基准评估进入实际智能体训练。

4. DPO 家族替代 PPO 做通用对齐#

DPO 及变体（KTO、IPO、SLiC-HF、Iterative/Online DPO）已在通用对齐中大量替代 PPO-based RLHF，但在线 RL（PPO/GRPO）仍在推理训练中占主导。

5. 基础设施收敛#

verl/HybridFlow 成为领先的开源 RL 框架，支持所有主要算法并扩展到 671B 参数。vLLM 集成快速 rollout 已成为所有框架的标准。大规模异步 RL（AReaL、SRL）突破万卡级别。

6. 多智能体从博弈走向 LLM#

从 AlphaStar 的博弈多智能体到 Anthropic 的并行 Claude 编译器，多智能体正在从传统 MARL 向 LLM-based 多智能体系统演进，Melting Pot 等评估套件也在适配 LLM 场景。
