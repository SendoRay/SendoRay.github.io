最近在给推理框架接入一个新的采样算子（Min-P Sampling）时，复盘了一下自己的开发节奏，发现一个比较稳定的"四步法"非常好用。这篇博客不聊采样算法本身的数学细节，而是把它当成一个载体，记录一下我现在做新功能时的工程方法论。 TL;DR：先理解背景 → 再看历史方案 → 写 minimal test 跑通新功能 → 接入大框架。每一步都要有明确的退出条件，不要跳步。 为什么需要方法论？ 做推理框架这种"大型系统中的小改动"，最容易踩的坑不是算法本身，而是： 上来就在大框架里改代码，跑一次端到端要好几分钟，debug 链路超长； 不了解既有方案，自己造了一个和已有 API 风格冲突的轮子； 改完之后才发现假设错了，要回退一大片； 接入后某个边界条件挂了，但因为没有最小可复现用例，定位非常困难。 我现在的做法是把整个开发过程拆成四个阶段，每个阶段产出一个最小的、可验证的交付物，然后再往下走。 Step 1：详细分析背景 这一步的目标是回答清楚三个问题： 我到底要做什么？——把需求翻译成一句可以贴在 PR 描述里的话。 它在系统里的位置在哪？——是新增一条独立路径，还是替换/扩展已有的某个模块？ 成功的标准是什么？——什么样的测试通过、什么样的指标达标，才算"做完了"。 以 Min-P 采样为例，这一步我会写下来类似这样的笔记： 需求：在现有采样器中新增 Min-P 采样策略，作为 Top-K / Top-P 之外的可选项。 位置：推理框架的 Sampler 模块，紧跟在 logits 后处理之后、multinomial 采样之前。和现有的 Top-K / Top-P 是兄弟关系，不替换它们。 成功标准： 给定一组固定的 logits 和 min_p 阈值，输出的候选 token 集合可被手工验证； 与已有的 Top-K / Top-P 可以叠加使用，行为符合预期； 接入后端到端跑一个小模型，开启 Min-P 不会触发任何已有测试回归。 这一步的关键不是写得多漂亮，而是把模糊的需求收敛成可以验证的条件。如果连成功标准都说不清楚，后面的所有工作都是没有方向的。 Step 2：弄清楚之前是怎么做的 我以前的习惯是看完需求就开始动手，结果经常出现两种尴尬： ...

大模型的商业竞争趋势

原文来自知乎 机智的Ryan 自 ChatGPT 发布那一天起，我就时常思考大语言模型的商业规律和竞争动态，但一直找不到一个好的抓手。 前两天，DeepSeek V4 发布。我写了一篇长文，讨论 V4 的一些决策错误。忽然间，三年来想不通的商业问题，全部串起来了。 本系列文章将讨论大语言模型的商业模式、竞争规律、定价策略，并做出一些判断，方便读者未来验证我此刻的理论。 一、学习曲线 首先，如何理解大语言模型的商业竞争规律？ 我认为，它完全可以使用半导体产业的逻辑去思考。模型的训练，等于半导体的研发。模型的推理，等于生产。 那么，半导体是如何决定公司胜负的？ 其核心规律在于 learning curve，也就是学习曲线。这一点我在 V4 那篇文章中短暂提过，现在完整阐述。 学习曲线的逻辑非常简单。一件事，你做得越多，就会做得越好。 然而，其商业竞争的本质，很多人没有思考过。因为做得越多，就会做得越好，那么做得越好，就会带来更好的模型和产品。 在质量提高的同时，成本也会降低。因此，用户会更喜欢用你的产品。用户多了，营收自然也会更多。营收增加之后，研发资金随之增加，你就可以去研发下一代更好的产品，进入一个正向循环。 当这样的正向循环持续几十年，竞争对手就会一个一个掉队，失去参与下一代竞争的资格。 当这样的竞争持续几十年，专业化程度不断提高，竞争者数量就会自然减少。因为每一代产品的开发成本都会不断上升，资金和人才都会越来越集中。 当大部分厂家发现，哪怕投入如此多的研发成本，仍然无法在市场上取得有效地位，也无法分摊高昂的研发成本时，它们就会自然退出。 以芯片制造为例。台积电在 1987 年创立的时候，还没有专业化晶圆代工这个概念。很多半导体厂都是自己生产自己设计的芯片，因此有能力制造芯片的厂非常多，可能有上百家。 然而，随着制程不断缩小，技术难度不断提升，到 2000 年左右，能够生产先进芯片的就只有十家左右。到 2010 年，这个数量进一步减少到四五家。 台积电在 2018 年量产 7 纳米，随后在 N7+ 节点引入 EUV。此刻，全世界还有希望跟上的就只有三星和英特尔。 然而又过了几年，到了 2 纳米节点，每一代制程的固定投资成本高达 300 亿美元以上。英特尔和三星也都扛不住了。于是，我们就来到了今天台积电独大的局面。 实际上，我们可以说，凡是大众化的产品，最后的市场格局都会陷入类似情况。 比如 GPU，就是英伟达独大。TPU 和 AMD 只能占第二、第三的份额，而 AMD 往后就几乎可以忽略不计。 同时，毛利润也会随着市场排名而变化。英伟达可以收取高达 70% 的毛利润，第二名可能收取四五十个百分点。从第三名开始，制造 GPU 就变成一门亏本的生意，因为量太小，不足以分摊研发成本。 内存是这样，工业软件也是这样。许多行业都是这样。 我认为，大模型的发展将完全依照半导体的发展规律。这就意味着，能否占据最大的市场份额，对一家企业来说是生死攸关的。 假如你的营收足够多，你就可以用营收中的一部分作为投资。当你的投资高于其他公司时，你的大模型训练就会越好，推理成本就会越低。 因此，大家就会更愿意用你的模型，于是带来营收增长，进入一个正向循环。 倘若你的市场规模不够，就无法筹集到足够的资金来支持下一代研发，逐渐掉队。 这应该是一个浅显的道理。然而，包括我在内，大部分人都会被各种干扰因素影响判断。 比如在短期内，大模型的研发成本依然主要通过融资实现，不管是股权投资，还是某种形式的借债。 然而，如果大模型要成为一项持续的生意，它终有一天会回到以营收，或者说以推理利润带来的正向现金流，来支持研发的阶段。 如果你相信 AGI，相信它真的会给全人类创造巨大的财富，那么它所产生的收益，以及它所对应的投资成本，都会高于现有大部分行业。 ...
